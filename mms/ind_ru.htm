MMS-компилятор

[ Скачать | Обсуждения ]
MMS (англ. Multimedia Messaging Service) — это служба мультимедийных сообщений, являющаяся эволюцией SMS. Она поддерживает текст с мультимедийными изображениями, анимацией, звуком и видео. Данный контент отображается и воспроизводится в приложении для обмена сообщениями получателя.

MMS поддерживается большинством мобильных телефонов, выпущенных с конца 2001 года, включая базовые модели и смартфоны: Android, Symbian, Blackberry, Windows Phone и другие. Apple поддерживает MMS начиная с iOS 3, кроме SMIL-форматирования. То есть Apple использует фиксированные шаблоны форматирования вместо тех, которые указаны в содержимом MMS-файла. Некоторые устройства поддерживают MMS с разметкой HTML.

Технические спецификации MMS изначально были созданы WAP Forum совместно с 3GPP. В настоящее время спецификации поддерживаются международной организацией Open Mobile Alliance (OMA). SMIL — это язык, используемый для настройки отображения MMS. SMIL для MMS — как HTML для веб-страниц. Большинство приложений для MMS частично поддерживают SMIL 2.0. Более поздние версии SMIL не получили широкого распространения в мире MMS.

MMS-компилятор — это инструмент с открытым исходным кодом, который можно использовать для создания файлов Multimedia Messaging Service (MMS) на основе шаблонных данных: текста, изображений, аудио, видео. Этот инструмент можно использовать в Windows или Linux (x86). Он не требует установки дополнительных компонентов.

Ознакомьтесь с Обзором и Руководством пользователя для более подробной информации и практических рекомендаций по созданию MMS-файлов, совместимых с большинством мобильных телефонов.

При публикации MMS-файлов на веб-сервере должны быть соблюдены следующие условия:

  • URL-адрес должен быть доступен через APN для MMS, настроенный в мобильном телефоне. Если доступ заблокирован оператором, временным решением является настройка APN для Интернета в качестве APN для MMS, чтобы мобильный телефон мог загружать MMS-файлы с любого общедоступного веб-сайта. Этот вариант следует использовать только для тестирования, так как разрешение на загрузку и воспроизведение MMS-файлов с любого веб-сайта может быть небезопасной.
  • MIME-тип для файла MMS должен быть application/vnd.wap.mms-message
  • Некоторые клиенты MMS не поддерживают HTTP-перенаправления. Поэтому не используйте сокращатели URL или аналогичные инструменты.
  • Ранние клиенты MMS не поддерживают SSL/TLS. Поэтому URL должен использовать обычный HTTP, а не HTTPS.

Большинство телефонов не отображают никаких ошибок, если загрузка MMS не удалась. Поэтому может потребоваться использовать отладчик, например, adb для Android, чтобы увидеть точные сообщения об ошибках.
